Attributes : number and balance, both are private (-) 
o These should be private to enforce encapsulation, ensuring that they cannot

6.5. Provide examples from your sketch and one sentence explanations for the following OO 
notions: encapsulation, abstraction, inheritance, polymorphism, aggregation/composition. [10 
marks] 
 
Encapsulation: The Account class has private attributes like balance and number, with 
public methods deposit() and withdraw(). 
Encapsulation protects sensitive data by restricting direct access and allowing controlled 
modifications through public methods. 
 
Abstraction: The Account class could have been abstract, to allow for other account types to 
be created without replicating internal structure and logic common to all. 
Abstraction hides the internal complexities and shows only relevant parts. 
 
Inheritance: Current Account and Savings Account both inherit from the Account class. 
Inheritance allows these subclasses to reuse common features from Account while 
implementing their own specialised behaviours. Inheritance is a way of using abstraction.  
 
Polymorphism: The withdraw() method behaves differently in Current Account (which 
might allow overdrafts) than in Savings Account (which might restrict withdrawals). 
Polymorphism allows a single method to have different implementations depending on the 
specific class. 
 
Aggregation/Composition: represented as aggregation example, allows construction of 
bigger entities from parts, as shown with Bank (as a system) being an aggregation of ATM, 
Account, Customer.

# 设计上下文 / Design Context

## Users
- 主要使用者：站点拥有者本人 LZQ（Bristol CS 在读，期末复习场景）
- 使用情境：考试前每天反复练题；早上专注、深夜暗光环境都有
- 任务：选择 / 填空 / 论述三种题型 + 背题 / 答题两种模式 → 高强度认知工作

## Brand Personality
- 三个词：**克制 · 精致 · 沉静**（restrained · crafted · quiet）
- 不是「酷炫科技」、不是「Material 学院派」，是「桌上一台 iMac、咖啡见底、专注阅读」的氛围
- 整体接近 Apple Newsroom / Apple Support / Linear 三者的中间地带

## Aesthetic Direction
- **流派**：Apple-refined（参考 SF Pro 的字体节奏、苹果产品页的留白、iCloud / Apple Music for Web 的层级处理）
- **不是**：iOS 拟物大圆角 / 霓虹紫渐变 / Emoji 满屏 / Material 卡片堆叠 / 学院风
- **主题**：light / dark 智能切换（跟随系统 + 手动覆盖），同等精心调校
- **颜色策略**：暖灰中性色调（stone-tinted neutrals）+ 单一克制蓝色 accent，避免多色斑斓

## Design Principles
1. **字体即设计语言**：用 Bricolage Grotesque（display，带可变 opsz 轴）+ Geist（body / UI）+ Geist Mono（数据）三足，不依赖装饰物
2. **OKLCH 色彩 + 60-30-10 视觉权重**：中性色撑场，accent 只在关键交互点出现（按钮、状态、激活态）
3. **发丝边框 > 大投影**：1px hairline + 微弱 inset highlight，不要 2024 通用风的 `shadow-xl`
4. **节奏感胜过密度**：通过留白和字体粗细对比建立层级，不靠尺寸堆砌
5. **Apple-style 单一动效系统**：ease-out-quart 200-280ms，所有过渡共享同一节拍
6. **Light/Dark 不是默认值**：每种主题独立调色，dark 模式用 graphite 暖灰而非纯黑

## Scope
当前应用：`/questiongen` 整套界面（顶栏、3 种题卡、过渡动效、主题切换）。
其他模块（厨房 emerald、博客 purple-dark、塔罗 mystic）保持各自主题不动。
设计 token 全局可用，未来其他页面可按需采纳。
